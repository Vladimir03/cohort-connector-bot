## 2. Веб-админка в preview

**Контекст:** Python-бот и FastAPI-админка запускаются одним процессом внутри dev-сэндбокса на `WEB_PORT` (8000). Preview Lovable показывает только TanStack-фронтенд проекта на другом порту — напрямую отдать `http://localhost:8000` в preview-iframe нельзя, у sandbox-порта Python нет публичного URL.

**План:**
1. Убедиться, что Python-процесс бота жив, и поднять FastAPI-админку на `0.0.0.0:8000` (она уже встроена в `bot.py` через `asyncio.gather`).
2. Проверить `http://localhost:8000/health` изнутри сэндбокса (`curl`) — подтвердить, что web и bot отвечают.
3. Так как Lovable preview не маршрутизирует на произвольный порт Python-процесса, открыть админку через временный публичный туннель (`cloudflared`/`localhost.run` через ssh), вернуть пользователю публичный URL и пароль (`admin` / значение `ADMIN_PASSWORD` из `.env`).
4. Указать в ответе: URL временный (живёт пока работает сэндбокс), для постоянного доступа нужен деплой контейнера на свой хост.

### Открытый вопрос

Lovable preview-iframe не умеет показывать произвольный сервис из Python-сэндбокса. Подтверди, какой вариант подходит:

- **A.** Поднять временный публичный туннель (cloudflared/ssh -R) к `localhost:8000` и прислать ссылку — быстро, но URL временный и сэндбокс может перезапуститься.
- **B.** Ничего не туннелировать: я дам команды для локального запуска (`docker run` или `python bot.py`) на твоей машине / VPS, и ты сам откроешь `http://<host>:8000`.
- **C.** Перенести админку на TanStack-фронтенд проекта (читает ту же `sas_bot.db` через серверные функции) — тогда она реально откроется в Lovable preview, но это уже отдельная задача переписывания UI.
